02)  Introdução:

 

Existem casos em que é necessário estabelecer algumas verificações lógicas de condições definidas sucessivamente.

 

A partir do momento em que uma determinada ação é executada, ela pode também levar a outras condições, de forma que não haja limites.

 

Dessa forma existe a possibilidade de usar uma condição dentro de outra condição, o que leva a uma estrutura de decisão encadeada ou aninhada.

 

Exemplo 1 (se ... então) (Homogênea):

 

se (condição1) então

  se (condição2) então

    se (condição3) então

      se (condição4) então

        comando1;

      fimse; // Bloco condição 4

    fimse; // Bloco condição 3

  fimse; // Bloco condição 2

fimse; // Bloco condição 1

 

 

 

Exemplo 2 (se ... então ... senão) (Homogênea):

 

se (condição1) então

  comando1;

senão

  se (condição2) então

    comando2;

  senão

    se (condição3) então

      comando3;

    senão

      se (condição4) então

        comando4;

      fimse;

    fimse;

  fimse;

fimse;

 

 

 

Exemplo 3 (Heterogênea):

 

se (condição1) então

  se (condição2) então

    comando1;

  fimse;

senão  

  se (condição3) então

    comando2;

  senão

    se (condição4) então

      se (condição5) então

        comando3;

      senão

        comando4;

      fimse;

    fimse;

  fimse;

fimse;

 

 

 

Exemplo 4 (decisão encadeada múltipla – escolha):

 

escolha (variável)

  caso valor 1 :

    comando 1;

 

  caso valor 2 :

    comando 2;

 

  caso valor 3 :

    comando 3;

 

  senão

    comando 4;

fim_escolha;
